# Hooks Configuration

このフォルダに含まれる `hooks.json` は、旧 Claude Code 環境で使用されていた「自動化スクリプトの定義集」です。

## ⚠️ 重要：Antigravity での動作について
**このフォルダにファイルを置くだけでは、Antigravity 上でフックは動作しません。**
Antigravity は現在、この JSON 形式の設定ファイルを自動的に読み込んで実行する機能を持っていません。

**このファイルの目的：**
「以前どのような自動化を行っていたか」を確認するための**参照用資料（レシピ集）**として残しています。

---

## JSON ファイルの読み方

`hooks.json` に記載されている内容は、以下のような意味を持っています。これらを参考に、Antigravity のワークフロー（`/workflows`）や手動コマンドとして再現することができます。

```json
{
  "hooks": [
    {
      "matcher": "git push",             // ① きっかけ（トリガー）
      "hooks": ["open editor for review"] // ② やること（アクション）
    }
  ]
}
```

### 各項目の意味
- **matcher (トリガー)**:
  - どのコマンドやツールが使われた時に反応するか。
  - 例: `"tool == \"Edit\""` → 何かファイルを編集した時
- **command / hooks (アクション)**:
  - 実行されるシェルコマンドやスクリプト。
  - 例: `grep -n 'console.log' ...` → 編集したファイルに console.log が残っていないか検査する

## どうやって使うの？（活用例）

この JSON を見て「あ、以前は git push の前にチェックしてたんだな」と気づいたら、Antigravity では以下のように代替します。

1. **手動で行う**:
   - 作業が終わったら、意識的に `grep` コマンドなどでチェックを行う。
2. **ワークフローを作る**:
   - `.agent/workflows/pre-push-check.md` のようなファイルを作り、その中に「git push する前にコードレビューを行う」という手順を定義し、コマンドとして実行できるようにする。
